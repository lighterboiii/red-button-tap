# Buttoff

Монорепозиторий: фронт и бэк в **разных папках** — деплой каждой части отдельно.

- `frontend/` — React, TypeScript, [FSD](https://feature-sliced.design/): `app`, `pages`, `widgets`, `features`, `entities`, `shared`.
- `backend/` — Node.js (Express), `POST /api/tap` — взвешенный дроп редкости + текст.

## Локально

Терминал 1:

```bash
cd backend && npm install && npm run dev
```

Терминал 2:

```bash
cd frontend && npm install && npm run dev
```

База API для фронта: **`frontend/.env`** — `VITE_API_URL` (полный URL бэкенда). В **режиме разработки**, если не задано, запросы идут на **`http://localhost:3001`** (или `VITE_API_PORT`, если бэк на другом порту). Прокси Vite (`/api` → 3001) можно не использовать — клиент ходит на бэкенд напрямую (CORS в Express включён).

Для **продакшена** задайте `VITE_API_URL` в сборке (URL бэкенда со схемой, без слэша в конце).

**Vercel:** в проекте фронта → Settings → Environment Variables → `VITE_API_URL` = `https://<твой-бэкенд>` (Railway, Render, Fly и т.д.), затем **Redeploy**. Без этой переменной запросы идут на тот же домен, что и фронт (`/api/...`), и дают **404**, потому что на Vercel только статика. См. `frontend/.env.example`.

## Telegram (Mini App + бот)

1. Создай бота в [@BotFather](https://t.me/BotFather), возьми **токен**.
2. В BotFather: **Bot Settings → Menu Button** или **/newapp** — укажи URL фронта (HTTPS), где открывается это приложение.
3. В бэкенде скопируй `backend/.env.example` в `backend/.env`, вставь `TELEGRAM_BOT_TOKEN=...`.
4. Пока токен **не** задан, `POST /api/tap` и `GET /api/me` работают без Telegram (локальная разработка в браузере).
5. С токеном бэкенд проверяет подпись `initData` из [Telegram Web App](https://core.telegram.org/bots/webapps): фронт шлёт заголовок `Authorization: tma <initData>` (см. `frontend/src/shared/api/client.ts`).

**Локально без туннелей:** в `backend/.env` добавь `SKIP_TELEGRAM_AUTH=1` — API не требует `initData`, даже если `TELEGRAM_BOT_TOKEN` задан (удобно тыкать на `localhost`). В проде этот флаг **не включай**. Опционально `DEV_MOCK_TELEGRAM_USER_ID=123` для стабильного id в `/api/me`.

Аватар в «Персонаж» берётся из `photo_url` пользователя Mini App.

## Механика (база)

Каждый тап запрашивает исход на сервере: **обычное** чаще всего, **редкое** и **легендарное** — с низким шансом. В UI показывается приблизительный процент для выпавшей редкости, чтобы ожидание «прикола» было понятным.

## Бой (атака, защита, крит)

Источник правды — **бэкенд**: `backend/src/combat/` (`constants.js`, `computeStats.js`, `formulas.js`, `resolveBattle.js`). Фронт только показывает ответы `POST /api/combat/preview` и `POST /api/battle`.

### Статы

- **Атака / защита** на экране — сумма вкладов с **надетых** предметов (редкость × слот; расчёт на сервере).
- **Крит** — сумма: крит с экипировки (с капом) + бонус с тапов (с капом), итог не выше капа из `constants.js`. В бою: бросок «сработал крит» с этим шансом.
- **HP игрока** перед боем: `PLAYER_HP_BASE + защита × PLAYER_HP_PER_DEFENSE` (защита — суммарная с экипировки).

### Каждый раунд

1. **Твой удар** (сначала ты):
   - **Размах** — случайный множитель от `BATTLE_SWING_MIN` до `BATTLE_SWING_MAX` (например 0.88–1.12) — имитация разброса силы удара.
   - **База** = `твоя_атака × размах`.
   - Если выпал **крит** — сила после крита = `база × PLAYER_CRIT_DAMAGE_MULTIPLIER` (сейчас ×2).
   - **Плоская броня врага** = `защита_врага × PLAYER_HIT_ENEMY_DEFENSE_FACTOR` (сейчас 35% от защиты врага).
   - **Урон** = `max(1, ⌊сила_после_крита − плоская_броня⌋)`, не больше текущего HP врага.

2. **Удар врага** (если враг жив):
   - **Сила** = `атака_врага × размах` (свой бросок размаха, крита у врага нет).
   - **Твоя броня** = `твоя_защита × ENEMY_HIT_PLAYER_DEFENSE_FACTOR` (сейчас 38% от твоей защиты).
   - **Урон по тебе** = `max(1, ⌊сила − твоя_броня⌋)`, не больше текущих HP.

3. Лимит раундов — `BATTLE_MAX_ROUNDS`; если лимит исчерпан — победитель по оставшемуся HP (см. код).

Перед боем клиент запрашивает **противника** (`POST /api/battle/opponent`), показывает статы, затем стартует бой с тем же `enemy` в `POST /api/battle`. В журнале — кратко: **урон**, **сколько сняла броня**, при критe — пометка **«Крит!»**.

### Тренировка vs случайный бой

Та же формула; у тренировочного манекена фиксирован профиль (`SPAR_OPPONENT`). Тренировка **не** тратит прочность экипировки.
